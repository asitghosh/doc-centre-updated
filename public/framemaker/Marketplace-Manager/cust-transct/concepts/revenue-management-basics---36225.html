Understand your orders ledger

Navigation path

Manage > Channel > Marketplace > Billing | Orders

Your orders ledger shows an almost-live representation of orders as they occur, rendered separately at the product level and at the product edition level. The ledger does not refresh automatically in your browser, so its representation of orders is only as recent as when your browser loaded it.

Product

The product-level view renders seven columns in its UI version and 27 columns in its exported CSV version.

    • The UI version is searchable by:
      • order cost — when the query string is any number of digits preceding a decimal point, which then precedes any number of digits.
      • order ID — when the query string is a sequence of no more than 18 contiguous digits.
      • company name
      • company external ID
      • product name

 

    • The UI version supports filtering by:
      • Status — The order state. Any one of these: ACTIVE , CANCELED , FAILED , FREE_TRIAL , FREE_TRIAL_EXPIRED , INITIALIZED , or SUSPENDED.
      • Cost — PAID or FREE.
      • Created between — when the order was created. A date range.
      • Pending (checkbox) — whether to include orders in any of the PENDING states.

 

    • The UI version supports sorting by the ascending or descending values in any of these columns:
      • ID — the subscription ID
      • Created — the order creation date
      • Status — the order state
      • Company — the name of the billable user’s associated company
      • Product — the product name

 

The UI version of the Orders ledger does not support sorting by values in either the Frequency column or the Total Fee column. Its platform-default sorting order is by descending values in the ID column.

Edition

The product edition-level view renders eight columns in its UI version and 25 columns in its exported CSV version.

    • The UI version is searchable against values in these columns:
      • ID — the order ID.
      • Company — the name of the billable user’s associated company.
      • Product — the product name.
      • Total Fee — the invoice amount per billing cycle.
    • The UI version supports filtering by:
      • Status — The order state. Any one of these: ACTIVE , CANCELED , FAILED , FREE_TRIAL , FREE_TRIAL_EXPIRED , INITIALIZED , or SUSPENDED.
      • Cost — PAID or FREE.
      • Created between — when the order was created. A date range.
    • The UI version supports sorting by the ascending or descending values in any of these columns:
      • ID — the order ID.
      • Created — the order creation date.
      • Status — the order state.
      • Company — the name of the billable user’s associated company.
      • Product — the product name.
      • Edition — the product edition name.
      • Frequency — the billing cycle interval between invoices.
      • Total Fee — the amount due per invoice.

Understand order details

Navigation path

Manage > Channel > Marketplace > Billing | Orders > { order }

Clicking any row in the orders ledger causes a modal popup to open, wherein we describe the corresponding order in massive detail.

At a high level, these details describe the buyer, the billing amount and frequency, the consumption of allocated seats, the order creation date and contract expiration date, the next invoice date, the order history, the assigned users, the overall order history, and more.

Understand your invoices ledger

Navigation path

Manage > Channel > Marketplace > Billing | Invoices

Your invoices ledger shows an almost-live representation of invoices as your marketplace issues them. (The ledger does not refresh automatically in your browser, so its representation of invoices is only as recent as when your browser loaded it.) It renders five columns in its UI version and 14 columns in its exportable CSV version.

    • The UI version is searchable against these values:
      • the billed user’s first name, last name, and email address
      • the name and ID of the billed user’s associated company
      • an invoice number (when the query string is digits only and its length ranges from 1 to 18 characters long)
      • an amount due (when the query string is any number of digits offset by one decimal point from any other number of digits)

 

    • The UI version supports filtering by:
      • an invoice state ( PAID , UNPAID , or CARRIED )
      • an invoice creation date range

 

    • The UI version supports sorting by the ascending or descending values in any of these columns:
      • ID — the invoice ID.
      • Date — the invoice creation date
      • Customer — the name of the billed user’s associated company
      • Total — the amount due
      • Status — PAID , UNPAID , or CARRIED

Understand invoice details

Navigation path

Manage > Channel > Marketplace > Billing | Invoices > { invoice }

Clicking any row in the invoices ledger causes a modal popup to open, wherein we describe the corresponding invoice in detail. At a high level, these details describe the billed party, the billable product or service, the billable amount, any payments remitted thus far, and any remaining balance.

In addition, clicking the PDF button ( ) in the Invoice Details modal causes your marketplace to generate a downloadable — and printable — invoice upon demand. This PDF includes exactly the same details that the Invoice Details modal includes.

Understand your payments ledger

Navigation path

Manage > Channel > Marketplace > Billing | Payments

Your payments ledger shows an almost-live representation of payments as your billing system receives them. (The ledger does not refresh automatically in your browser, so its representation of payments is only as recent as when your browser loaded it.)

Overview

The overview-level view — via this toggled state above the table: — cites the combination of all successful and unsuccessful payments, excluding only those payments that applied a 100 percent discount. The Overview view renders seven columns in its UI version and 17 columns in its exportable CSV version.

    • The UI version is searchable against these values:
      • payment ID
      • customer
      • vendor
    • The UI version supports filtering by:
      • payment method
      • payments versus refunds
      • payment attempt outcome


 

Details

The detail-level view — via this toggled state above the table: — cites only the line items from successful payments. This excludes failed payments. The Details view renders eleven columns in its UI version and 35 columns in its exportable CSV version.

    • The UI version is searchable against these values:
      • payment ID
      • customer
      • vendor
    • The UI version supports filtering by:
      • payment method
      • payments versus refunds
      • invoice line type
      • whether a 100 percent discount was applied
      • when your marketplace processed the payment (or refund)

Understand payment details

Navigation path

Manage > Channel > Marketplace > Billing | Payments > { payment }

Clicking any row in the payments ledger’s Overview view (or any Payment ID link in the same ledger’s Details view) causes a modal popup to open, wherein we describe the corresponding payment in detail. At a high level, these details describe the billed party, the payment itself, and every invoice that it applies to.

In addition, two UI buttons facilitate refunds when all or part of the described payment is disputed or was erroneous.

    • Refund Payment — Opens the Refund Payment modal popup. Here, after you choose which invoiced amounts to refund, and you submit the form:
      • Your marketplace backend makes API calls to your payment gateway and commands that it process the configured refund as instructed.
      • Your marketplace records are updated to show the payment as refunded.
    • Enter Manual Refund — Opens the Enter Manual Refund modal popup. Here, after you choose which invoiced amounts to refund, and you submit the form, your marketplace records are updated to show the payment as refunded.

Understand your quotes ledger

Navigation path

Manage > Channel > Marketplace > Billing | Quotes

Your quotes ledger shows an almost live representation of all saved quotes for your marketplace. Sales agents initialize quotes for consumers at one or the other of these UI paths:

    • reseller

Manage > Reseller > Reseller | Users > { user } > [ Make Purchase ] > { product } > [ Start Purchase ] > [ Save as Quote ]
    • sales support representative

Manage > Channel > Home | Users > { user } > [ Make Purchase ] > { product } > [ Start Purchase ] > [ Save as Quote ]

A quote has two possible states: PENDING PURCHASE and COMPLETED.

The ledger does not refresh automatically in your browser, so its representation of quotes is only as recent as when your browser loaded it. When a refreshed view no longer includes a quote that was previously visible, it means that the corresponding quote was either completed or deleted in the interim.
