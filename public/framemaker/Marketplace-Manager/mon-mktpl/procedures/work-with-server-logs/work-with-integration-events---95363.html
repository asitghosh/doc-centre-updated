Work with integration events

Navigation path

Manage > Channel > Marketplace > Events | Event Logs > ( Integration Events table)

Before you begin

    • Log in to your marketplace as its administrator.

Procedure

    1. Step 1. Go to Manage > Channel > Marketplace.

      2. Step 2. Click through Events | Event Logs. Then, scroll to the Integration Events table.

      Step 3. ( Optional ) Click Show Filters to expand a toolbar whose controls help you to filter which event records the table should show. Then, do any or all of the following.

      NOTE: Filter selections take effect immediately, and each effect is additive.
        • Do you want to see log entries from a specific time frame?

      (Optional) Filter events by when they occurred

          1. a. Define the start and end of a date range by entering dates manually in the paired date-entry fields.

            OR
            Click a field’s corresponding calendar icon (
        ) to expose the date-picker widget for populating that field.

      NOTE: When you enter dates manually, use the calendar date notation syntax that is standard in your locale and which your marketplace is configured to use — either M/D/YY ) or D/M/YY.
        • Do you want to limit by severity level which events the table should list?

      Choose one or more event severity levels

          1. a. ( Optional ) Check the Show Green checkbox to list events that were triggered at the INFO level.

            2. b. ( Optional ) Check the Show Yellow checkbox to list events that were triggered at the WARN level.

            c. ( Optional ) Check the Show Red checkbox to list events that were triggered at the ERROR level.
        • Do you want to limit the table to list events of only one type?

      Set just one event type to list

          1. a. Choose an option from the Event Type list.
        • Do you want to show or hide events that were resolved manually?

      Set how the table should handle manually resolved events

          1. a. Choose an option from the Manual Resolution list.
        • Do you want to show or hide asynchronous events while they are pending?

      Set how the table should handle pending asynchronous events

          1. a. Check or clear the Only Show Asynchronous Pending Events checkbox.
        • Do you want to show or hide the events that pertain to published products?

      Set how the table should handle published products

          1. a. Choose an option from the Published App list.
        • Do you want to show or hide events that pertain to users with the tester user role?

      Set how the table should handle events from testers

          1. a. Choose an option from the Tester User list.

      Step 4. ( Optional ) Do any of the following:

  • Click a name in the Application column to view details about the corresponding product.
  • Click a hash in the Event Type/Event Token column to view the same XML message that the product’s vendor received for the cited event.
  • Click a value in the User/Company column to view details about the corresponding user account.

Step 5. ( Optional ) Click Download CSV.

Step 6. Stop. You have completed this procedure.
