Understand API encryption key pair editing

Navigation path

Manage > Channel > Settings > Integration | API Clients > Create API Client

An API system integration to your AppDirect-powered marketplace is possible when each system that underlies the outside service has its own, unique key pair for data encryption purposes.

NOTE: The initiating party — the one that sends a request — is considered the API client. Do not confuse the API client with his or her REST client software for composing and sending API requests.

You can generate, edit, regenerate, or revoke any number of these key pairs upon demand and associate them with the necessary permissions, authentication types, and return URLs for integrated systems. Basic key management is essential to maintaining the security of your API integrations.

TIP: You can read our authentication documentation for API developers at htps://docs.appdirect.com/marketplace-manager/apis/introduction/authentication.

Understand data migration

Navigation path

Manage > Channel > Settings > Integration | Migration > Create Dataset

NOTE: Before you can migrate data, you must pay for AppDirect migration service. Otherwise, all UI elements for the data migration feature set are turned Off.

You can upload well-formed datasets and merge them into the database tables that describe various entities, relationships, and states in your marketplace. The supported data set types are:

    • Users
    • Companies
    • Memberships
    • Reseller Associations
    • Discount Codes
    • Company Entitlements (meaning, subscription states)
    • User Entitlements (meaning, seat assignments)
    • Microsoft Customers
    • Microsoft Subscriptions
    • Microsoft Domains
    • Google Accounts

TIP: To learn about the conventions that define what is valid or not valid in a migration data set, see the read-only sample spreadsheet in the Upload Files area at Manage > Channel > Settings > Integration | Migration > [ Create Dataset ] > [ Download ].

Understand webhooks

Navigation path

Manage > Channel > Settings > Integration | Webhooks

While APIs wait for, and respond to, client requests before transmitting a given dataset, webhooks transmit data to their clients in something close to real time, bypassing the requirement for client requests. Specified client systems receive refreshed data on configured endpoint URLs as soon as events on the host system create, change, or delete records of a given type.

You can configure your AppDirect-powered marketplace to:

    • Share information with its webhook client systems following the creation, modification, or deletion of any company, discount, invoice, membership (company-user association), order, payment, payment Instrument, quote, reseller, sales lead, sales opportunity, subscription, or user.
    • Authenticate itself to its webhook client systems via OAuth 2, via tokens, or not at all.

Notification body

Webhook notifications include the expected resource payload but also show a link to the API URL where you can submit an HTTP GET request to receive the same content object payload. For example, your marketplace could send a notification like the following when a sales opportunity is created:

{

"uuid": "8241cbe0-40da-4673-91e5-abf16c063d86",

"timestamp": 1432311484,

"resource": {

"type": "SALES_OPPORTUNITY",

"uuid": "0729adc2-fc66-40ac-a59d-9a3f81dd4c69",

"url": "http://api.example.com/api/v2/crm/reseller/opportunity/0729adc2-fc66-40ac-a59d-9a3f81dd4c69

"content": {

"uuid": "0729adc2-fc66-40ac-a59d-9a3f81dd4c69",

"externalIdentifier": "some_external_id",

...

}

},

"resourceAction": "CREATED",

}

Consider the following.

    • Each notification includes a "content" object as a child of the "resource" element. Its value is identical to what you would receive in response to an HTTP GET call to the value of the "url" element.
    • Each notification "type" — such as SALES_OPPORTUNITY or RESELLER_REGISTRANT — contains a "content" object that is relevant to it. Correct interpretation of a "content" object’s value can occur only in relation to the same notification’s "type".
