Feature aggregation and navigation to configure marketplace settings

Table 2-1 is a quick reference to all marketplace configuration features of Marketplace Manager, exposing which functions we cluster together and citing exactly where in the UI you can find each function.

Table 2-1: Feature navigation to configure your marketplace


Manage > Channel > Settings

Settings |

General Settings

Use UI elements and features here, optionally, to do any or all of the following:

    • [ Launch setup guide ] — Start a configuration wizard that guides you through the initial setup of your marketplace.
    • [ Edit homepage layout ] — Open a layout editor to adjust the look and feel of your product listing pages.
    • About — Set your company and marketplace names. Upload a marketplace logo.
    • Email Settings — Configure email address usage.
    • Support Resources — Define customer contact points for end-user technical support.
    • Legal URLs — Enter the URLs for your marketplace legal terms and privacy policy.
    • Analytics — Enter a unique tracking ID for use with Google Analytics.

Settings |

Marketplace Functionality

Use UI elements and features here, optionally, to do any or all of the following:

Marketplace Password — Isolate your marketplace during downtimes, such as site development or testing, by imposing a sitewide password. (When enabled, the sitewide password is separate from, and prerequisite to, your user account login form.)

Reviews & Questions — All of these:

    • Set prerequisites for the visibility of user-generated products reviews.
    • Set behavioral parameters for user-generated product reviews, ratings, and questions.
    • Show or hide all product-specific questions.
    • Enforce manual approvals for all user-generated content before publication.

Customer UI — All of these:

    • Enforce telephone number collection during user account activation.
    • Show or hide all details about obtaining product-specific technical support.
    • Show or hide user and company profile pages.
    • Enable company administrators to import applications obtained from outside your marketplace.
    • Show or hide all product-specific support details sitewide.

Customer Roles — All of these:

    • Set all newly registered users to receive the developer user role by default.
    • Set all newly registered users to receive the company administrator user role by default.
    • Send copies of all billing-related email messages to their associated sales agents, if any, when the reseller role or the sales support representative role is enabled.

Security — All of these:

    • Load a CAPTCHA during user account registration.
    • Define a complexity policy for user account passwords.
    • Allow or forbid customer support staff to request impersonation access.
    • Add OpenID trusted realms.

SEO — All of these:

    • Enter a metadata description of your marketplace, which search engines can expose in search results.
    • Enter metadata keywords to influence the weight that search engines attribute to your marketplace.
    • Edit your robots.txt file.

Custom UI |

FAQs

Use UI elements and features here, optionally, to do any or all of the following:

    • Create and name any number of sorting categories for FAQs. Or, alternatively, remove categories.
    • Add or remove any number of FAQs in any sorting category.

Customer Notifications

See Chapter 4, Notifications.

Custom Form Fields

Use UI elements and features here, optionally, to do any or all of the following:

    • Create, edit, or delete UI form fields to collect non-standard company attributes. For example, you might want to collect a company’s tax identification number or its stock market ticker symbol.
    • Create, edit, or delete UI form fields to collect non-standard user attributes. For example, you might ask each user to specify his or her job title.
    • Specify which user roles on your marketplace are authorized to see a custom field’s recorded value.
