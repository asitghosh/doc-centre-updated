The purpose of setting a return address

Navigation path

Manage > Channel > Settings > Settings | General Settings

You must set the return address that generated event notification messages show when their recipients use email client software to read them. For example, you might set the return address to notifications@example.com , where example.com is your organization’s email domain.

To b e clear, your entry here is not the value that any parameter injects when you use the parameter in a notification template. Rather, your marketplace uses the supplied address for the reserved purpose of sending SMTP (Simple Mail Transfer Protocol) transmissions with valid email headers.

When events transpire, your marketplace sends notifications to the registered users who its events effect.

TIP: We recommend that the return address should be valid and, and that any responses to it should be read.

The purpose of setting a BCC address

Navigation path

Manage > Channel > Settings > Settings | General Settings

You can set a “blind carbon copy” (BCC) address to receive a copy automatically of each notification message that your marketplace sends, encompassing all event types and all message recipients.

Consider carefully whether populating this address is consistent with your organization’s goals and principles. The person who, or team that, receives these messages could become privy to sensitive information.

The purpose of CCing sales agents on assisted sale notification messages

Navigation path

Manage > Channel > Settings > Settings | Marketplace Functionality

You can configure your marketplace to send sales agents exact copies of the same notification messages that beneficiaries of their assisted sales receive. These messages are specific to the corresponding products, their contracts, and their payments.
