Work with encryption key pairs for API service and system integrations

Before you begin

    • Log in to your marketplace as its administrator.

Navigation path

Manage > Channel > Settings > Integration | API Clients

Procedure

    1. Step 1. Go to Manage > Channel > Settings.

      2. Step 2. Click past Integration | API Clients.

      Step 3. Do one of the following.

        • Do you want to create a key pair?

      Create a key pair

          1. a. Click Create API Client.

      The API Client Settings popup opens. In it, the OAuth checkbox is checked by default.

            b. Enter a name for this API client.

      TIP: We recommend that you name it for the integrated service.

            c. Choose an option from the Permissions list.

        1. Read Only — The integrated system is authorized to receive data from your marketplace but cannot create, change, or delete marketplace data.
        2. Read and Write — The integrated system is authorized to receive, create, change, and delete marketplace data.

            3. d. ( Optional ) Check the OAuth 2.0 checkbox to support modern authentication for the integrated system and expose additional UI elements.

        3. Check one or more checkboxes in the OAuth 2.0 Grant Types area, as appropriate:
      • Authorization Code
      • Implicit
      • Password
      • Client Credentials
      • Refresh Token
    2. Populate the OAuth 2.0 Return URL field.

        3. e. Click Save Settings.

      The new API client is created, assigned a consumer key, and appears in the API Clients table on the API Clients page.
        • Do you want to edit a key pair?

      Edit a key pair

          1. a. Click the gear icon (
        ) next to an API client whose key pair should be edited.

      The clicked gear icon exposes a list of supported options.

            b. Choose Edit Settings from the list.

      The API Client Settings popup opens. It shows the Consumer Key and Consumer Secret fields, whose respective values are not directly editable. It also shows the API client’s editable name, permissions, and authentication type.

            c. Edit whichever entries are no longer valid.
        • Do you want to regenerate the API client’s shared secret?

      Regenerate a shared secret

          1. a. Click the gear icon (
        ) next to an API client whose key pair should be edited.

      The clicked gear icon exposes a list of supported options.

            b. Choose Regenerate Secret from the list.

      The Regenerate Consumer Secret popup opens. It shows the consumer key pair that is currently in effect.

            c. Click Regenerate Consumer Secret to generate a new secret for this API client’s key pair.

      A message above the table confirms that the new secret was generated.
        • Do you want to revoke the API client’s key pair?

      Revoke a key pair

          1. a. Click the gear icon (
        ) next to an API client whose key pair should be edited.

      The clicked gear icon exposes a list of supported options.

            b. Choose Revoke Access from the list.

      The Revoke Access popup opens.

            c. Click Revoke Access to deactivate the key.

      A message above the table confirms the access was revoked for the key. Revoked clients cannot be re-enabled, but you can create a new client with the same name as a revoked client.

      Step 4. Click Save Settings.

      Step 5. Stop. You have completed this procedure.
