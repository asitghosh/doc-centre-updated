Work with migration data

Before you begin

    • Pay for the AppDirect migration service.
    • When ad-blocking or popup-blocking software is installed on your client system, add your marketplace server to the whitelist.
    • Log in to your marketplace as its administrator.

Navigation path

Manage > Channel > Settings > Integration | Migration

Procedure

    1. Step 1. Go to Manage > Channel > Settings.

      2. Step 2. Click past Integrations | Migration.

      Step 3. Do one of the following.

        • Do you want to create a migration dataset?

      Create a dataset

          1. a. Click Create Dataset.

      The Data Migration page opens.

            b. ( Optional ) Click Download to obtain a sample migration data file, which you can examine to learn about structure and syntax conventions for each type of migratable record.

            c. Enter a name in the Batch Name field, which pertains correctly and equally to each of the migration files you plan to upload in this session. You must enter a name.

      There are multiple upload forms — one apiece for each of the record types that you can migrate.

        • Users
        • Companies
        • Memberships
        • Reseller Associations
        • Discount Codes
        • Company Entitlements (meaning, subscription states)
        • User Entitlements (meaning, seat assignments)
        • Microsoft Customers
        • Microsoft Subscriptions
        • Microsoft Domains
        • Google Accounts

        d. Do the following.

      1. Click Choose File in each form that corresponds to data which you have processed and ready for migration.

        2. When the File Upload window opens, browse through your client system’s mounted volumes and folder structure until you find the appropriate file for a given form.

        Click the data file to select it, and then click Open.

        Repeat these steps as many times as needed to complete your dataset.

        e. Click Upload Dataset.
        • Do you want to review a migration report?

      Review a migration report

          1. a. Browse through the Data Migration table until you locate the dataset whose migration details matter to you.

            2. b. Click Download Report in the same table row that cites the dataset’s name.

            c. Open the report in your preferred spreadsheet software and check any entries in the Errors column.
        • Do you want to download product login credentials?

      Download product credentials

          1. a. Browse through the Data Migration table until you locate the dataset whose migration details matter to you.

            2. b. Click Download Credentials in the same table row that cites the dataset’s name.

            c. Open the credentials file in your preferred spreadsheet software to review it.

      Step 4. Stop. You have completed this procedure.
