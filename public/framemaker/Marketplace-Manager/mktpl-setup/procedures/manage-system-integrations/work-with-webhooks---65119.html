Work with webhooks

Navigation path

Manage > Channel > Settings > Integration | Webhooks

Before you begin

    • Log in to your marketplace as its administrator.

Procedure

    1. Step 1. Go to Manage > Channel > Settings.

      2. Step 2. Click past Integration | Webhooks.

      Step 3. Do one of the following.

        • Do you want to create a webhook?

      Create a webhook

          1. a. Click Add Webhook.

      The Add Webhook popup opens.

            b. Enter a client system’s endpoint URL, which should receive data from your marketplace, in the URL field.

      NOTE: Webhook endpoints must support and use HTTPS connections.

            c. Choose an entity from the Resources list to transmit records for entities of that kind. The types are:

        1. Company
        2. Discount
        3. Invoice
        4. Membership (company-user association)
        5. Order
        6. Payment
        7. Payment Instrument
        8. Quote
        9. Reseller Registrant
        10. Sales Lead
        11. Sales Opportunity
        12. Subscription
        13. User

      Your choice in the Resources list constrains which checkboxes are available to check under the Action label. The checkboxes that you might see include:

        • Added
        • Changed
        • Deleted

        d. Check one or more checkboxes under the Action label to specify which system events should trigger notification messages about the specified resource.

        e. Click a button under the Auth Type label to set if and how your marketplace server should authenticate itself to the specified client endpoint.

        f. Click Save Webhook.
        • Do you want to edit a webhook?

      Edit a webhook

          1. a. Click the gear icon (
        ) next to a defined webhook that should be edited.

            b. Choose Edit Webhook from the list.

      The Edit Webhook popup opens.

            c. ( Optional ) Edit a client system’s endpoint URL, which should receive data from your marketplace, in the URL field.

      NOTE: Webhook endpoints must support and use HTTPS connections.

            d. ( Optional ) Choose an entity from the Resources list to transmit records for entities of that kind. The types are:

        1. Company
        2. Discount
        3. Invoice
        4. Membership (company-user association)
        5. Order
        6. Payment
        7. Payment Instrument
        8. Quote
        9. Reseller Registrant
        10. Sales Lead
        11. Sales Opportunity
        12. Subscription
        13. User

      Your choice in the Resources list constrains which checkboxes are available to check under the Action label. The checkboxes that you might see include:

        • Added
        • Changed
        • Deleted

        e. ( Optional ) Check one or more checkboxes under the Action label to specify which system events should trigger notification messages about the specified resource.

        f. ( Optional ) Click a button under the Auth Type label to set if and how your marketplace server should authenticate itself to the specified client endpoint.

        g. Click Save Webhook.
        • Do you want to delete a webhook?

      Delete a webhook

          1. a. Click the gear icon (
        ) next to a defined webhook that should be deleted.

            b. Choose Delete Webhook from the list.

      The Delete Webhook popup opens. It:

        • warns you that deleting the specified webhook prevents your marketplace server from sending records to the defined endpoint.
        • cites the endpoint URL, the resource, and the event.

        c. Click Delete.

Webhook notifications include the expected resource payload but also show a link to the API URL where you can submit an HTTP GET request to receive the same "content" object payload. For example, your marketplace could send a notification like the following when a sales opportunity is created:

{

"uuid": "8241cbe0-40da-4673-91e5-abf16c063d86",

"timestamp": 1432311484,

"resource": {

"type": "SALES_OPPORTUNITY",

"uuid": "0729adc2-fc66-40ac-a59d-9a3f81dd4c69",

"url": "http://api.example.com/api/v2/crm/reseller/opportunity/0729adc2-fc66-40ac-a59d-9a3f81dd4c69

"content": {

"uuid": "0729adc2-fc66-40ac-a59d-9a3f81dd4c69",

"externalIdentifier": "some_external_id",

...

}

},

"resourceAction": "CREATED",

}

Consider the following.

      • Each notification includes a "content" object as a child of the "resource" element. Its value is identical to what you would receive in response to an HTTP GET call to the value of the "url" element.
      • Each notification "type" — such as SALES_OPPORTUNITY or RESELLER_REGISTRANT — contains a "content" object that is relevant to it. Correct interpretation of a "content" object’s value can occur only in relation to the same notification’s "type".

      Step 4. Click Save Settings.

      Step 5. Stop. You have completed this procedure.
